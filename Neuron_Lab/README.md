## 07: Получился класический перцепторн без функции активации

В лаборатории функция активации, выход из программы и защита от неверного символа, как по мне здесь это не нужно. Цикл (while) может уйти в бесконечность. Всё через (CTRL + C). 

- Логика работы перцептрона температура + колличество воды.

- Для контроля нагрева реализована формула предыдущего значения.

- **Зачем:** Это нужно для определения колличества воды. По скорости изменения температуры перцептрон понимает сколько воды.

- **Вход х2:** Это (sk_nagrev) Сюда подаётся скорость нагрева, которая вычисляется через предыдущее значение плюс буфер.

- **Проблемы** Первый шаг цикла (while) выход (output) **прыгает** некоректный выход.

- **Почему без активации** Если  **(out < 1)** чайник вкл. Иначе выкл. 

- **Тест** Температура 1 это 10°С. Виртуальный нагрев 10°С, 20°С Скорость нагрева ровна единице, Резкий рост нагрева 50°С  Скорость нагрева 3.0 чайник выкл. При плавном росте изменения температу скорость 6агрева ровна единице до 100°С.

## 06: Обучить нейрон для температуры и колличество воды

- Контроль температуры и колличество воды (наличие воды) в чайнике.
- Один нейрон два входа (x1),(x2). 
- x1 - температура воды.
- x2 - скорость нагрева.
Все данные через (scanf). В начале обучаем. Потом в бесконечном цикле (while) вводим данные температуры и скорость нагрева.
